Version: Genesis

Status: Foundational

# Purpose

The Information Architecture defines how knowledge, products, services, and ideas are organized across the FYNLY ecosystem.

Every page should have one clear purpose.

Navigation should feel natural.

Nothing should exist without reason.

# Design Rule

Every page should answer exactly one primary question.

If a page attempts to answer several unrelated questions,

split the page.
